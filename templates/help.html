전체 업무 흐름

배송비 단가 산정은 아래 순서로 진행합니다. 최초 1회 마스터 데이터를 세팅한 후, 이후에는 ③~⑤만 반복하면 됩니다.

1
마스터 데이터 등록 차량 단가 · 이고비 · 거점 변동용차 · 거리별 단가표 · 자사 센터를 먼저 등록합니다.
2
고객사 등록 및 기준 데이터 입력 고객사를 등록하고 상품(PLT 환산 기준)·점포 마스터를 업로드합니다.
3
출고 내역 업로드 배송비 산정 화면에서 엑셀 형식의 출고 내역 파일을 업로드합니다.
4
배송비 산정 실행 직송/공동배송 분류 → 이고비 + 변동용차비 자동 계산이 이루어집니다.
5
결과 확인 및 내보내기 산정 결과를 검토하고 엑셀로 내보내 견적서 작성에 활용합니다.

화주 상세 분석

특정 화주사의 물류비를 항목별로 분석합니다. 화주 현황에서 화주명 클릭 시 진입합니다.

{% if screenshots.get('analytics_detail') %} {% else %}
스크린샷 미생성
{% endif %}
  • 총 물류비 / 직송 / 공동배송 / 이고비 / 변동용차비 5개 카드로 항목별 금액 표시
  • 각 카드에 총 BOX·PLT, 일평균 BOX·PLT 정보 포함
  • 이고비 상세 버튼 → 거점별 이고비 내역 모달
  • 변동용차비 상세 버튼 → 거점별 변동용차비 일자별 내역 모달
  • 직송/공동배송 전환 임계값(PLT) 조정 슬라이더 — 실시간 비용 변화 프리뷰
Tip
변동용차비 카드의 일평균은 지도 시뮬레이션과 동일한 알고리즘(planRoutes)으로 계산됩니다. 카드 총합 = 일평균 × 공동배송 영업일수입니다.
